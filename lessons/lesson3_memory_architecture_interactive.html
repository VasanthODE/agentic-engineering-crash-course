Memory architecture

The model is stateless — it forgets everything between calls. Yet real agents must remember: a long conversation, a knowledge base, a user across months. Memory architecture is deciding what to keep, where, and how to put it back in the window at the right moment.

01 Seconds → hours

Session memory

02 Static knowledge

Vector retrieval (RAG)

The highest-leverage memory architecture you'll build. You have far more knowledge than fits in a window — so for each question, retrieve only the handful of chunks you actually need, by meaning.

Embed once, retrieve always, ground every answer. Training knowledge becomes the reasoning engine; your retrieved chunks become the facts. That separation is what makes agents useful for real businesses. But RAG remembers knowledge, not a specific user — that's next.

04 Minutes · computation

Prompt caching

Different in kind from the others — it remembers computation, not content. Your stable prefix is identical every call, yet the model re-processes it from scratch each time. Caching makes you pay that quadratic cost once.

The cost math
A prompt splits into a stable prefix (same every call) and a fresh suffix (changes each call):
The order matters: stable content must come first, because caching works on a prefix. On the next call, if that prefix is byte-for-byte identical, the model skips re-processing and starts fresh only at the suffix. Anything that changes (the question, retrieved chunks) goes after the cached prefix — one changed token at the start invalidates the whole cache.

Profitable from the second call onward. It ties straight to O(n²): processing a 5,000-token prefix is quadratic in attention, and caching lets you pay that once instead of every call. Caches expire in minutes — so it suits high-frequency repeated-prefix workloads.
The architecture: a stateless model persists information four ways. Session memory carries this conversation (resend, slide or summarise) · RAG carries knowledge (embed once, retrieve by meaning, ground) · episodic memory carries the user (extract, store by identity, reload) · prompt caching carries computation (reuse the stable prefix). The model never remembers — you reconstruct the right context at the right time, on the right clock.
