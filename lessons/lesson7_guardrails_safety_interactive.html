01 The structure

The three boundaries

The agent is a powerful core surrounded by untrusted edges. A guardrail is a checkpoint on every arrow crossing a boundary — input, output, and action. Click each layer.

Defense in depth, fail closed. No single check is perfect, so layer independent checks (PII is watched on BOTH boundaries on purpose). And when a guardrail is uncertain or errors, default to BLOCK — a guardrail that fails open disappears exactly when you need it.
02 The entry boundary

Input guardrails

Everything that runs on the user's message before the model sees a token. Reject or sanitize bad input at the door — it's cheaper to refuse a malicious request than to clean up after the agent acts on it.

03 The exit boundary

Output guardrails

Everything that runs on the model's response before the user sees it. The last line of defense — because the model is non-deterministic, even a clean request can produce a leak, a hallucination, or an off-policy answer.

You can't prevent every bad output, so you inspect every output. Block, regenerate (with feedback), repair/redact, or escalate. The grounding check enforces Lesson 3's RAG promise; PII is checked here too — defense in depth.
04 Spans both boundaries

PII handling

The concern that lives at both edges, and carries real legal weight. Data minimization: the model rarely needs the actual data — strip it early, restore it only where strictly required.

The model is a system you don't trust with secrets, so you give it none. Regex catches structured PII (card, Aadhaar, PAN); NER catches names. Tokenize so a leak exposes a meaningless token, not real data — shrinking the blast radius to one guarded path.
05 The most consequential boundary

Human-in-the-loop

Where the agent stops talking and starts doing. A bad response can be caught on the way out; a bad action, once executed, may be impossible to undo. Which actions can the agent take alone, and which need a human?

The gate is enforced by your guardrail, not the model. The agent emits a normal tool call; the boundary intercepts it. On reject, the reason returns as an observation and the agent REPLANS — the human is a checkpoint in the ReAct loop, not a replacement for it.
